# Data Visualization & Dashboards

A separate zone with its own posture. Where the project has a dashboard, admin panel, or analytics: charts, metrics, tables are **maximally animated, detailed, alive** — not a "boring table". A full, worked dataviz.

## Posture

- Everything animated and detailed — live, real-time data: counters, updating charts.
- Any number animates in via the counter (the number-counter rule from `motion.md` and `storytelling.md` carries here).
- Metric cards, live-updating graphs — the dashboard feels alive, not a static dump.

## Tables

Not a default browser table. Worked: clear hierarchy, fixed-width numerals for numeric columns, sensible row separation (alternating fill over heavy borders), animated sort indicator, hover states.

## Real-time

Live counters and updating graphs where the data is genuinely live. Numbers tick; graphs redraw smoothly (`transform`/`opacity`, 60fps).

## Status

- The whole dataviz posture (animated, detailed, alive) where a dashboard/admin/analytics exists: **MUSTHAVE-DEFAULT**.
