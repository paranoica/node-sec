# Pattern: Features

This file gives Claude what to assemble, not what to copy.

## 1. Purpose
Communicate what the product does / why it is worth it — broken into digestible units.

## 2. Block composition
**Required:** a section heading, 3–6 feature units (each: a title + a short description).
**Optional:** a per-feature icon or mini-visual, a per-feature link, one highlighted "hero feature", a supporting visual per unit.

## 3. Order logic
Section heading sets the frame → feature units follow. Order units by importance, strongest first. If one feature is the headline capability, give it more visual weight (size, a richer visual) — do not flatten all units to the same card.

## 4. Variants by mode
- **Clean** — even grid of calm units, consistent rhythm.
- **Statement** — varied unit sizes (a bento-style break of the grid), one unit much larger, asymmetry, scroll-reveal stagger.

## 5. Technique bindings
- List stagger / cascade on entrance (`principles/motion.md`, `principles/storytelling.md`).
- Layout: even grid (Clean) vs grid-break / overlap (Statement) — `principles/layout.md`.
- Icons follow the one-set rule (`principles/icon-morphing.md`).

## 6. Typical mistakes
- 6 identical cards with no hierarchy — the definition of boring.
- Icons from mixed sets / weights.
- Every unit the same size when one feature clearly outweighs the rest.

## 7. The hook in this section
Carry the site's hook through this section too, not just the hero (`principles/concept.md`): state per build how it advances the one central idea, or how it stays deliberately quiet so the hook lands elsewhere.
