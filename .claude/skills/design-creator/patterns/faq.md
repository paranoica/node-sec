# Pattern: FAQ

This file gives Claude what to assemble, not what to copy.

## 1. Purpose
Answer the recurring objections / questions that block a decision — compactly.

## 2. Block composition
**Required:** a section heading, a list of question/answer pairs (accordion).
**Optional:** category grouping, a search box, a "still have questions" contact link.

## 3. Order logic
Heading → Q/A list, ordered by how common / decision-blocking each question is — the most pressing first. FAQ usually sits late on the page, just before the footer, after the visitor has the context to have questions.

## 4. Variants by mode
- **Clean** — a simple accordion list.
- **Statement** — bolder type, smooth expand with content fade, possibly category tabs with a sliding indicator.

## 5. Technique bindings
- Accordion expand/collapse animated (`principles/motion.md`).
- Plus ↔ cross / chevron rotation via Level-1 icon morphing (`principles/icon-morphing.md`).
- Category tabs (if any): sliding active indicator (`principles/motion.md`).

## 6. Typical mistakes
- Accordion that hard-snaps open with no animation.
- All questions expanded at once — defeats the compactness.
- An icon that does not change with the open/closed state.

## 7. The hook in this section
Carry the site's hook through this section too, not just the hero (`principles/concept.md`): state per build how it advances the one central idea, or how it stays deliberately quiet so the hook lands elsewhere.
