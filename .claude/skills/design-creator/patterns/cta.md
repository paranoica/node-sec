# Pattern: CTA section

This file gives Claude what to assemble, not what to copy.

## 1. Purpose
A focused conversion moment — one clear action, minimal distraction. Usually the last push before the footer.

## 2. Block composition
**Required:** a short compelling headline, a primary CTA button.
**Optional:** a one-line supporting sentence, a secondary CTA, a reassurance line (no card needed / free / cancel anytime), a background visual.

## 3. Order logic
Headline → supporting line → CTA → reassurance. Deliberately sparse — every extra element dilutes the single action. This section should feel like a clearing, not a busy block.

## 4. Variants by mode
- **Clean** — centered, generous air, one button.
- **Statement** — large type, a living background or 3D accent, an expressive button, possibly full-bleed.

## 5. Technique bindings
- CTA button: hover + active together, border-fill or slide-fill (`principles/motion.md`).
- Optional living background (`principles/decorative-graphics.md`) — must not outshine the button.
- Optional 3D accent (`principles/3d.md`).

## 6. Typical mistakes
- Two competing CTAs of equal weight — splits the decision.
- A busy section that buries the one action.
- A background effect louder than the button it surrounds.

## 7. The hook in this section
Carry the site's hook through this section too, not just the hero (`principles/concept.md`): state per build how it advances the one central idea, or how it stays deliberately quiet so the hook lands elsewhere.
