# References

Our own reference builds — exemplars Claude may safely learn from. These are **originals**, not copies of any brand's design files. They illustrate how the principles compose into a coherent whole at each mode.

A reference describes a concrete worked design with reasons — "why this works" stated in specifics (palette ladder, type scale, techniques applied), the same way `patterns/` files describe section anatomy.

- `clean-exemplar.md` — a full Clean-mode build.
- `statement-exemplar.md` — a full Statement-mode build.

Reference brands (Linear, Vercel, Locomotive, Active Theory, Igloo, etc.) may be looked at for *ideas* — character, technique vocabulary, anti-slop calibration. Their concrete design files are never copied: copyright, and cloning a single brand produces generic, derivative output. The engine extracts principles and applies them originally.
