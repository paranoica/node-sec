# Bedrock — обзор (RU)

Bedrock — это **репозиторий-шаблон** для GitHub, чтобы начинать программные проекты с качественным
результатом с первого дня. Ты клонируешь его, описываешь, что хочешь построить, и набор согласованных
**агентных скиллов** разворачивает спеку, план задач и правила, которым следует каждое изменение — а
затем держит работу проходящей через нужный гейт качества. **Скиллы исполняются в Claude Code**;
**проект, который они создают**, агент-агностичен — его правила в кросс-агентном **`AGENTS.md`**,
поэтому дальше Cursor, Codex, Windsurf и другие работают под тем же каноном.

## Что внутри

- **genesis** — проводит интервью и пишет *источник правды* проекта: решения, архитектуру, глоссарий
  и открытые вопросы (`docs/`), переразбираемый план задач (`PLAN.md` + `genesis.tasks.json`), правила
  проекта (`AGENTS.md` — кросс-агентный канон — плюс тонкую обёртку `CLAUDE.md`), первую структурную
  карту кода и реальный рабочий CI-workflow (`.github/workflows/`).
- **prompt-refiner** — тихо превращает мутный запрос («почини это», «сделай чтоб работало») в точный и
  маршрутизированный — но **только** когда запрос не взял ни один другой скилл.
- **design-creator** — проектирует и строит отличительный веб-интерфейс, спроектированный против
  генерического «AI-slop».
- **code-review** — жёсткий ревьюер существующего кода: безопасность / корректность / производительность.

## Чем отличается (преимущества)

- **Спека — источник правды, а план переразбирается из неё.** Меняешь решение — затронутые задачи
  помечаются автоматически. План никогда не «замороженный одноразовый», который правят руками.
- **Не выдумывает недостающие решения.** Всё, что ты ещё не решил, записывается как открытый вопрос, а
  не угадывается — значит спека не содержит тихой выдумки.
- **Ничто не врёт молча.** Карта кода несёт штамп свежести (устаревшая карта никогда не выдаётся за
  факт); связь спека↔задача проверяется хэшом; контракты самих скиллов проверяются на дрейф в CI.
- **Нужный инструмент запускается сам.** Дизайн → design-creator; аудит существующего кода →
  code-review; всё остальное → простой цикл по плану. Ты говоришь — файлы остаются синхронными.
- **Агент-агностичность by construction.** Правила живут в `AGENTS.md` — кросс-агентном стандарте,
  который нативно читают Cursor, Codex, Roo и Windsurf; Claude Code читает через тонкую обёртку. Один
  канон, все агенты (Antigravity — через `.agents/rules`-импорт).
- **Честно, дружелюбно к агентам и инспектируемо.** Всё — обычные файлы по документированным путям, а
  гейты качества имеют зубы: они **падают** (ненулевой код), а не просто предупреждают.

## Начать

Запусти `/genesis` в Claude Code и скажи, что хочешь построить (созданный проект дальше работает с
любым AGENTS.md-агентом). Все функции, как работает каждая часть и edge cases — в
**[INSTRUCTIONS.ru.md](INSTRUCTIONS.ru.md)**.
